# 인공게임

보안 사고의 이름, 표적, 징후, 피해 결과를 연결해서 이해하는 의뢰형 시뮬레이션 게임이다.  
실제 해킹 구현이 아니라, 사고 대응 판단과 인과관계 학습에 초점을 둔다.

## 현재 구조

- `server.js`: Node.js 정적 웹 서버
- `backend/api.py`: Python 게임 데이터 API
- `public/`: 로그인, 의뢰 보드, 체크리스트 기반 게임 UI

## 핵심 흐름

- 회원가입 / 로그인은 브라우저 `localStorage` 기반
- 로그인 직후 사건이 시작되지 않고 `의뢰 보드`가 먼저 열린다
- 회사별 의뢰를 하나 선택한 뒤 고정 체크리스트를 직접 체크해 제출한다
- 제출 후 약 10초 동안 처리되고 결과가 반영된다
- 계약 하나가 끝나면 다시 의뢰 보드로 돌아가 다음 회사를 고른다
- 사건과 회사 조합은 새 의뢰 세트마다 랜덤으로 섞인다

## 실행

### 1. Python API

```powershell
cd $HOME\Desktop\인공게임
python .\backend\api.py
```

### 2. Node 웹 서버

다른 터미널에서 실행:

```powershell
cd $HOME\Desktop\인공게임
npm start
```

### 3. 접속

`http://localhost:3000`

## 메모

- MySQL 연동은 제거했다
- NPC 이미지는 네모형 플레이스홀더 카드로 구성했다
- 도움말 `?` 버튼에 용어 설명과 참고 링크를 넣어두었다
